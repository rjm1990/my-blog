代码写得再好，沟通不好也是白搭。

## 🎯 为什么要重视沟通？

- 70%的项目失败是因为沟通问题
- 升职加薪不只看技术，也看影响力
- 良好的沟通能减少返工

## 💬 与不同角色的沟通

### 与产品经理

**常见冲突**
- 需求频繁变更
- 需求描述不清
- 不理解技术复杂度

**沟通技巧**

1. **提前参与**: 需求评审阶段就参与
2. **说人话**: 用业务语言，不用技术术语
3. **提供方案**: 不要只说做不了，要提供替代方案
4. **量化成本**: "这个改动需要3天"比"这个很难"更清楚

```markdown
<!-- ❌ 不好 -->
这个需求技术上实现不了。

<!-- ✅ 好 -->
这个需求如果要做，需要重构整个支付模块，
大概需要2周时间。如果不重构，我们可以先
实现A和B功能，能满足80%的需求，只需要2天。
你觉得哪个方案更好？
```

### 与测试

**常见冲突**
- 提测质量差
- Bug描述不清
- 优先级不一致

**沟通技巧**

1. **自测充分**: 提测前自己先测一遍
2. **提供信息**: Bug复现步骤、环境信息要清楚
3. **尊重测试**: 测试是在帮你把关质量
4. **及时修复**: Bug不要拖着不修

### 与老板/领导

**常见问题**
- 不了解你的工作价值
- 不清楚项目进度
- 期望不切实际

**沟通技巧**

1. **主动汇报**: 不要等老板来问
2. **说结果**: 老板关心的是结果，不是过程
3. **管理预期**: 承诺前先评估，承诺后要兑现
4. **提供选择**: 让老板做选择题，不是问答题

```markdown
<!-- ❌ 不好 -->
老板，项目要延期了。

<!-- ✅ 好 -->
老板，项目目前进度80%，但是遇到了XX问题。
我有两个方案：
A. 增加1个人，能按时上线
B. 延期1周，不需要加人
你倾向哪个方案？
```

## 🗣️ 通用沟通技巧

### 1. 倾听

- 不要急着打断
- 确认理解：复述对方的观点
- 问清楚：不懂就问

### 2. 表达

- 结论先行
- 结构化表达
- 用数据说话

### 3. 非暴力沟通

```
观察 + 感受 + 需要 + 请求

例：我看到这个功能改了3次（观察），
    我感到有点沮丧（感受），
    因为希望能有稳定的需求（需要），
    能否我们在需求评审时多花点时间确认？（请求）
```

## 📧 写作沟通

### 邮件/消息

1. **标题清晰**: [项目名] 简短描述
2. **开头说重点**: 第一段说结论
3. **分点罗列**: 用1.2.3.而不是一大段
4. **明确行动**: 需要对方做什么

### 文档

1. **有结构**: 标题层级清晰
2. **有例子**: 抽象概念配例子
3. **有图表**: 一图胜千言
4. **有版本**: 标注更新时间

## 🚫 沟通反模式

1. **假设对方知道**: 不要假设，要确认
2. **情绪化沟通**: 冷静后再沟通
3. **回避冲突**: 建设性的冲突是有益的
4. **背后议论**: 有问题当面说

## 💡 提升沟通能力的方法

1. **多练习**: 主动承担技术分享
2. **多写作**: 写技术博客、文档
3. **多复盘**: 反思沟通中的问题
4. **多学习**: 读沟通相关的书

### 推荐书籍

- 《非暴力沟通》
- 《关键对话》
- 《金字塔原理》
