好的文档能减少80%的重复沟通。

## 📝 文档的重要性

- 帮助新人快速上手
- 减少重复回答相同问题
- 作为知识沉淀
- 方便日后维护

## 🎯 好文档的要素

### 1. 清晰的结构

```markdown
# 项目名称
一句话介绍

## 快速开始
5分钟内能跑起来的最小示例

## 安装
详细安装步骤

## 使用文档
功能说明 + 示例代码

## API文档
接口详细说明

## FAQ
常见问题

## 贡献指南
如何参与项目
```

### 2. 可运行的示例

```javascript
// ❌ 不好
// 使用fetch获取数据
fetch(url)

// ✅ 好
// 获取用户列表
fetch('https://api.example.com/users')
  .then(res => res.json())
  .then(data => console.log(data));
```

### 3. 图文并茂

用流程图、架构图辅助说明，比纯文字效果好10倍。

```
用户 → API网关 → 认证服务 → 业务服务 → 数据库
```

### 4. 更新日期

每篇文档都要标注最后更新时间，过期的文档比没有文档更糟糕。

## 📋 文档类型与模板

### README模板

```markdown
# 项目名称

一句话描述项目是做什么的

## 功能特性

- 特性1
- 特性2

## 快速开始

\`\`\`bash
npm install project-name
\`\`\`

\`\`\`javascript
const project = require('project-name');
project.doSomething();
\`\`\`

## 文档

[详细文档](./docs/README.md)

## 贡献

欢迎PR！查看[贡献指南](./CONTRIBUTING.md)

## License

MIT
```

### API文档模板

```markdown
## 接口名称

**接口地址**: `POST /api/users`

**功能说明**: 创建新用户

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 是 | 用户名 |
| email | string | 是 | 邮箱 |

### 请求示例

\`\`\`json
{
  "name": "张三",
  "email": "zhangsan@example.com"
}
\`\`\`

### 返回示例

\`\`\`json
{
  "code": 200,
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com"
  }
}
\`\`\`
```

## 🔧 文档工具推荐

- **Swagger/OpenAPI**: API文档自动生成
- **VuePress**: 技术文档站点
- **Notion**: 团队知识库
- **语雀**: 中文友好的文档平台

## ⚠️ 文档维护

- **定期更新**: 每次发版检查文档
- **版本管理**: 文档和代码一起提交
- **收集反馈**: 文档底部添加反馈入口
- **过期标注**: 过期文档要明确标注

## 💡 文档写作技巧

1. **假设读者是新手**: 不要假设读者了解背景
2. **先结论后细节**: 先说是什么，再说为什么
3. **用例子说话**: 一个好例子胜过十句解释
4. **保持简洁**: 能用一句话说清楚的不要用两句
5. **及时更新**: 代码改了，文档也要改
